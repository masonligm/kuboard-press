# Kuboard v4.x 变更日志

## v4.1.0.0

发布日期： 2026-03-08

- 新特性：
  - SSE 机制（Server Side Event）
    - 在服务端 K8S 对象发生变化时，通过 SSE 机制将最新变化实时更新到页面端
      - Deployment 详情页
      - StatefulSet 详情页
      - DaemonSet 详情页
      - Pod 详情页
    - 支持单节点 Kuboard 部署（使用内存队列发布事件）
    - 支持多节点 Kuboard 高可用部署（使用 Redis pub/sub 在所有实例间发布事件）
    - 用户可以在浏览器端禁用 SSE，此时退回到定时刷新页面的模式获取页面的更新内容
  - 增加便捷操作
    - 批量重启
    - 批量调整副本数
    - 批量修改镜像版本
  - K8S 对象（例如 ConfigMap、Deployment、StatefulSet、DaemonSet 等）详情页增加 K8S 对象历史版本查询功能
    > （仅支持查询在 Kuboard 中执行的版本变更历史）
- 优化：
  - 列表页增加集群、名称空间新的筛选模式
  - 列表页创建对象时，默认选中当前过滤器中第一个选中的集群、名称空间
  - 容器组详情页面增加跳转到归属对象的链接
  - 升级到 OpenJDK 25.0.2
  - 完善操作审计查询功能
    - 增加查询条件：操作对象名称
    - 操作类型可以多选，可以清空查询条件
  - 登录用户均可访问 kuboard 名称空间的 http:metrics-scraper:8000 接口的 get 方法
  - 日志追踪界面可以暂停滚动
  - 优化缓存表查询性能
- 问题修订：
  - 下载日志时，进度条显示不准确的问题以及下载文件不完整的问题
  - 容器组列表页，当容器组包含多个容器时，必须先选择容器，才能进入日志、终端界面
  - 使用表单创建 DaemonSet 时，对象类型不正确的问题
  - 如果浏览器日期跟服务器日期出现不一致时，Kuboard 在一天中特定时段提示 `缺少授权文件` 的问题
  - 执行 cd.kuboard.cn 分组下的接口时，审计日志不能记录请求报文的问题
  - 当角色关联到空的用户组时，编辑角色权限失败的问题
  - 更新集群的 token 时，需要重启服务才能生效的问题
  - 工作负载编辑页面快速创建 PVC 时不显示保存按钮的问题
  - 普通用户在导出资源时不能选择其他对象的问题

## v4.0.0.0

发布日期： 2025-10-12

- 新特性：
  - 终端界面可以选择字符编码（当容器内部的编码方式不是 UTF-8 时，切换字符编码可以避免终端显示的乱码）

      - 文本文件编辑界面不能保存
  - 日志界面可以选择字符编码
    - 已知问题：由于 kubernetes pods/logs 接口返回的编码已经不是原编码格式，日志界面不能正常还原汉字的原编码。

- 优化：
  - 在用户列表页/详情页增加解锁账号的操作
  - 容器组列表页增加日志/终端按钮
  - Deployment/StatefulSet/DaemonSet 列表页增加日志/终端按钮
